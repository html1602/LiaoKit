# 核心概念

## 窗口（Window）
- 承载业务内容的容器，组合各类插件与消息组件
- 管理多窗口，以及窗口间的状态与路由

## 消息（Message）
- 文本、图片、文件、插件气泡等不同消息类型
- 统一的消息列表与渲染规则（长列表性能优化需留意）

## 插件（Plugin）
- 可插拔的能力模块，如进度、统计、时间线、操作栏等
- 通过事件与状态与窗口/消息交互，形成完整业务流程

## 适配器（Adapter）
- 封装与外部服务交互，如 AI/流式 SSE/上传等
- 关注返回协议与订阅事件，确保正确渲染与错误处理

## 状态管理（State）
- 会话/窗口/消息的统一状态约束与更新流程
- 保持可预测性与可测试性，避免“隐式副作用”

## 样式与主题（Style/Theme）
- 提供可配置的样式变量与主题扩展点
- 保持命名规范与影响范围，避免全局污染

## 无障碍（A11y）
- 语义标签、键盘操作、视觉对比度建议
- 在组件/插件文档中提供具体策略与示例
