# 头像设置说明

## 文件位置

您的 `my.png` 图片文件当前位于：`src/static/my.png`

## 静态资源配置方式

在Vue + Vite项目中，有两种配置静态资源的方式：

### 方式一：src目录下的静态资源（当前使用）
- **位置**: `src/static/my.png`
- **引用方式**: `new URL('../static/my.png', import.meta.url).href`
- **优势**: 
  - 图片会被Vite处理和优化
  - 支持文件名哈希，利于缓存管理
  - 构建时会检查文件是否存在
  - 适合项目内部资源

### 方式二：public目录下的静态资源
- **位置**: `public/my.png`
- **引用方式**: `/my.png`
- **优势**: 
  - 直接复制到输出目录，不经过构建处理
  - 适合外部资源或第三方资源
  - 路径简单直观

## 图片要求

- **格式**: PNG格式（推荐）
- **尺寸**: 建议 40x40 像素或更高分辨率的正方形图片
- **大小**: 建议小于 500KB

## 使用方式

1. **当前方式**：图片已在 `src/static/my.png`，无需移动
2. 启动开发服务器：`npm run dev`
3. 访问多窗口预览页面：`http://localhost:3008/window-preview`
4. 创建新窗口时，所有用户和客服头像都会显示为您的图片

## 替代方案

如果您更希望使用 `public` 目录：

1. 将文件从 `src/static/my.png` 移动到 `public/my.png`
2. 修改代码中的引用路径为 `'/my.png'`

## 注意事项

- **src/static方式**：文件必须存在，否则构建时会报错
- **public方式**：如果文件不存在，浏览器会显示图片加载失败图标
- 修改图片后可能需要刷新浏览器缓存才能看到更新
- 当前使用的 `new URL()` 方式在构建时会生成优化后的文件路径
