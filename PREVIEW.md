# LiaoKit 多窗口预览系统 🚀

## 快速启动

### 方法一：开发服务器运行

```bash
# 安装依赖
npm install

# 启动开发服务器  
npm run dev

# 访问预览页面
# - 主页：http://localhost:3000
# - 多窗口预览：http://localhost:3000/window-preview
```

### 方法二：直接访问

在ComponentsShowcase页面中，点击"🚀 查看完整多窗口预览演示"按钮。

## 预览内容

### 🎯 功能演示

1. **创建窗口**：点击"创建窗口"按钮，自动生成客服会话窗口
2. **模拟消息**：使用"模拟消息"功能，随机发送消息到窗口
3. **切换模式**：体验AI与人工客服模式的无缝切换
4. **布局模式**：尝试四种专业布局（自由、网格、层叠、平铺）
5. **设备切换**：切换桌面/移动模式，体验响应式设计

### 🖥️ 桌面端特性

- **多窗口管理**：同时管理多个客服会话
- **拖拽交互**：点击标题栏拖拽移动窗口
- **层级管理**：点击窗口自动激活并提升层级
- **最小化功能**：最小化窗口显示在底部栏
- **布局算法**：智能的窗口排列和位置计算

### 📱 移动端特性

- **标签页模式**：窗口以标签页形式展示
- **单窗口内容**：当前活跃窗口全屏显示
- **未读徽章**：标签页显示未读消息数量
- **触摸友好**：优化的移动端交互体验

### 💬 聊天功能

- **完整消息界面**：用户信息、消息列表、输入区域
- **智能回复**：AI/人工模式不同的回复风格
- **快速回复**：预设回复选项，提升效率
- **实时更新**：消息、状态、未读计数实时更新

## 技术亮点

## 开发说明

### 添加新功能
1. 在`WindowListPreview.vue`中添加控制按钮
2. 在控制面板中实现对应的处理逻辑
3. 更新状态面板显示新的统计信息

### 自定义窗口内容
1. 修改`WindowContent.vue`组件
2. 添加新的消息类型或界面元素
3. 实现对应的交互逻辑

### 扩展布局模式
1. 在`useWindowManager.ts`中添加新的布局算法
2. 更新布局模式选择器
3. 添加对应的CSS样式
